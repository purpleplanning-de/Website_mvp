# Architecture Decision Records (ADR)

Dokumentation wichtiger Entscheidungen im Projekt – technisch und organisatorisch.

## Format

Jede ADR hat:
- **Status**: proposed | accepted | deprecated | superseded
- **Kontext**: Warum stand diese Entscheidung an?
- **Entscheidung**: Was wurde entschieden?
- **Begründung**: Warum diese Option und nicht eine andere?
- **Konsequenzen**: Was folgt daraus?

## Dateibenennung

`NNN-kurzer-titel.md` – z.B. `001-knowledge-management.md`

## Zweck

ADRs bewahren das "Warum" hinter Entscheidungen – auch über Session-Grenzen und Device-Wechsel hinweg.
Neue Claude Code Sessions können ADRs lesen und haben sofort den gewachsenen Kontext.
